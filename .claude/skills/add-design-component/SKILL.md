# Skill — Adicionar componente

1. Confirmar o problema e a iniciativa.
2. Procurar componente existente.
3. Criar ou atualizar spec.
4. Registar ID, maturidade, anatomia, variações, estados, acessibilidade, tokens e plataformas.
5. Criar exemplo exclusivamente demonstrativo.
6. Atualizar `components.json`, catálogo, testes e release.
7. Não promover para `approved`.
