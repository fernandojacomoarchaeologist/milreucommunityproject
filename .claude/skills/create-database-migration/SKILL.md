# Skill — Criar migration

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Receber objetivo e critérios.
2. Classificar risco.
3. Criar migration com nome semântico.
4. Evitar SQL irreversível.
5. Adicionar teste pgTAP.
6. Executar reset e testes locais.
7. Documentar rollback.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
