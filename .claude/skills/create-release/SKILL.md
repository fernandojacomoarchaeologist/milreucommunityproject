# Skill — Criar release

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Listar alterações.
2. Listar migrations.
3. Listar testes.
4. Registar riscos e rollback.
5. Registar pendências.
6. Confirmar ausência de segredos.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
