# Skill — Create spec

## Entrada necessária

- problema ou necessidade;
- iniciativa relacionada;
- produto de destino;
- público principal;
- conteúdo ou dados envolvidos.

## Procedimento

1. Executar `scope-check`.
2. Ler `docs/specifications/SPEC_TEMPLATE.md`.
3. Verificar `VISUALIZATION_REGISTRY.md`.
4. Criar um identificador `SPEC-XXX` sem reutilizar números.
5. Preencher apenas informações suportadas.
6. Colocar questões em aberto no fim da spec.
7. Marcar a spec como `draft`.
8. Não implementar a funcionalidade na mesma etapa, salvo instrução explícita.
9. Não aprovar a própria spec.
10. Registar a visualização como `specified` apenas após a criação da spec.

## Perguntas obrigatórias quando aplicável

- Que iniciativa é responsável?
- O conteúdo é real, preliminar ou de exemplo?
- Há dados pessoais, direitos ou consentimento envolvidos?
- Que idiomas entram nesta versão?
- A visualização já foi aprovada?
- Onde os dados serão persistidos?
