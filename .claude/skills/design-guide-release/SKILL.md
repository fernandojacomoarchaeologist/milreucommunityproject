# Skill: design-guide-release

## Procedimento

1. Confirmar versão do sistema.
2. Validar links e caminhos locais.
3. Confirmar que exemplos estão marcados como demonstrativos.
4. Confirmar ausência de fotografias e dados reais não aprovados.
5. Verificar os quatro idiomas essenciais.
6. Verificar comparação Spectral/Archivo.
7. Testar teclado, foco, zoom e movimento reduzido.
8. Confirmar que o logótipo não foi inventado.
9. Executar o validador.
10. Atualizar release e estado de componentes.

## Publicação

A skill prepara a release, mas não escolhe URL, domínio ou política de acesso. Perguntar quando esses dados forem necessários.
