# Skill — Responder a incidente

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Parar deployments.
2. Preservar logs.
3. Classificar impacto.
4. Não improvisar SQL produtivo.
5. Preparar correção testada.
6. Documentar comunicação e encerramento.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
