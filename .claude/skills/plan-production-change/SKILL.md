# Skill — Planear alteração produtiva

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Nunca aplicar a alteração.
2. Reunir migration, impacto e dependências.
3. Preparar rollback.
4. Identificar backup necessário.
5. Criar ticket e checklist.
6. Solicitar autorização explícita para o workflow.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
