# Skill — Registar decisão visual

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Receber decisão humana.
2. Atualizar Gate A.
3. Atualizar decisões e issues.
4. Relacionar componente e release.
5. Não promover itens não revistos.
6. Gerar changelog.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
