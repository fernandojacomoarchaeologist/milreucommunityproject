# Skill — Preparar rollback

## Objetivo

Executar este procedimento de forma segura e auditável.

## Procedimento

1. Identificar migration e dados afetados.
2. Avaliar rollback versus forward fix.
3. Não apagar dados posteriores.
4. Testar fora de produção.
5. Exigir aprovação.
6. Documentar resultado.

## Saída obrigatória

- ações executadas;
- ações não executadas;
- ambiente;
- validações;
- riscos;
- próxima decisão humana.
