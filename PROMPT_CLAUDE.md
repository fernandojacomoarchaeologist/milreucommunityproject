# Prompt para o Claude

Utiliza o texto abaixo como prompt após adicionar este pacote ao repositório.

---

Estou a integrar o **Pacote 01 — Fundação, Governação e Escopo, versão 0.1.0**, no repositório do Projeto Comunitário de Milreu.

Antes de alterar qualquer ficheiro:

1. Lê integralmente:
   - `README.md`;
   - `PACKAGE_MANIFEST.md`;
   - `CLAUDE.md`;
   - `COPYRIGHT.md`;
   - `RIGHTS.md`;
   - `releases/PACKAGE_01_v0.1.0.md`.
2. Lê todos os ficheiros incluídos em `docs/`, `.claude/rules/` e `.claude/skills/`.
3. Analisa a estrutura atual do repositório e identifica se já existem ficheiros com os mesmos caminhos.
4. Não sobrescrevas nenhum ficheiro existente sem apresentar primeiro uma comparação semântica e um plano de fusão.
5. Não alteres o site preliminar, imagens, dados do acervo, JavaScript, CSS, HTML ou integração com Google Drive neste pacote.
6. Não cries schemas, base de dados, design system, portal, mapa, app Flutter ou novas visualizações ainda.
7. Não inventes factos, registos, iniciativas, direitos, consentimentos, traduções, contactos ou decisões.
8. Verifica se todos os MDs e códigos novos ou alterados contêm a declaração:
   `© 2026 Fernando Rodrigues de Jácomo. Produzido no âmbito do Projeto Comunitário de Milreu.`
9. Mantém as declarações de terceiros e os direitos dos conteúdos separados da autoria da estrutura do projeto.
10. Utiliza a skill `package-intake` para conduzir a integração.

Entrega primeiro um relatório de pré-integração com:

- versão e escopo compreendidos;
- ficheiros novos;
- ficheiros já existentes;
- conflitos;
- proposta de criação, manutenção ou fusão;
- decisões bloqueantes, caso existam;
- perguntas objetivas apenas sobre o que impedir a integração.

Se não houver bloqueios, procede à integração preservando os caminhos do manifesto. No final, entrega:

- ficheiros criados;
- ficheiros fundidos ou alterados;
- ficheiros mantidos sem alteração;
- conflitos não resolvidos;
- pendências não bloqueantes;
- confirmação de que o release foi registado;
- confirmação de que nenhum código, dado ou interface do protótipo foi alterado.

Não avances para o Pacote 02 sem uma instrução explícita.
