# Ciclo de vida do registo

## Estado editorial interno

`draft` → `preliminary` → `in-review` → `reviewed` → `approved`

Estados adicionais:

- `deprecated`;
- `withdrawn`.

## Estado de publicação

`not-assessed` → `review-required` → `approved` → `published`

Estados de bloqueio ou reversão:

- `blocked`;
- `unpublished`;
- `withdrawn`.

## Estado dos dados

- `sample`;
- `preliminary`;
- `reviewed`;
- `published`;
- `contested`;
- `withdrawn`.

Os três eixos não devem ser condensados num único campo. Um registo pode estar editorialmente revisto, mas com publicação bloqueada por direitos.
