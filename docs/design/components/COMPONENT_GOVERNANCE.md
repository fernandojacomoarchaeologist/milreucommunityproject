# Governação dos componentes

Uma alteração exige problema documentado, spec, impacto nos tokens, acessibilidade, plataformas, testes e release. Apenas uma decisão registada pode promover um elemento para `approved`.

Componentes não devem incorporar lógica de publicação ou persistência; devem representar explicitamente os estados fornecidos pelos domínios correspondentes.
