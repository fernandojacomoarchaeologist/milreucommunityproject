# Modelo de componentes

Cada componente resolve um problema específico e possui: ID estável, categoria, maturidade, anatomia, variações, estados, contrato de acessibilidade, tokens, plataformas e exemplo demonstrativo.

A existência no catálogo não implica aprovação final. `validated` significa que o contrato está suficientemente definido para integração e testes.
